# Architecture Decision Records

Each ADR captures one significant decision: the context, the choice, the
alternatives considered, and the trade-offs accepted.

## Convention

*   One file per decision, named `NNNN-kebab-case-title.md` (zero-padded, sequential).
*   Numbers reflect decision order and are never reused.
*   Status is one of: `Proposed`, `Accepted`, `Superseded by ADR-NNNN`, `Deprecated`.
